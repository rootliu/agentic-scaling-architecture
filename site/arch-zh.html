Reference Architecture

三层可扩展 Agentic Runtime

面向生产级 AI agent 的参考架构 —— A = ⟨S, H, X⟩，外挂一个栈外数据子系统 𝒟。核心主张：agent 的逻辑扩展、物理扩展与数据扩展可以解耦，解耦点是显式契约层（Harness）。

本页是面向第一次接触者的导览：先走过三个层，再看每一层里到底装了什么，最后讲清楚 agent loop 如何运行 —— 并给出让它成为“研究主张”而非“工程图”的可证伪命题。

1三个层 — 每一层是干什么的

一个 agent 由三层叠成。Skills 说明要做什么（任务手册）；Harness 决定怎么把活干成（把意图翻译成可执行步骤，并掌管记忆、上下文、工具、代码生成、安全与审计）；Scaffold 是在哪里安全地跑（带 shell、网络、存储与模型 serving 的隔离 sandbox）。

关键在于：每一种增长加在不同的轴上 —— 加一个 Skill，多一种能做的任务（逻辑扩展）；加一个 Scaffold 实例，多一份吞吐（物理扩展）。因为它们只在 Harness 契约处相遇，所以互不拖累。外部数据则在旁边的子系统 𝒟 上沿第三条轴增长。

2每一层里装了什么 — Spec 清单

每一层都由可落地、可构建的部件组成。我们刻意把它们对齐到业界已有的格式，使本架构成为“扩展层”而非新协议：Skills 采用 Anthropic 的 SKILL.md 格式；Harness 的能力对象以 MCP tool schema 与 Claude Code 风格工具为底座；Scaffold 的 spec 覆盖隔离以及企业 SSO 与云集成。

对工程实现有两点值得强调。其一，code generation（Read/Edit/Bash + test/lint）与 tool synthesis（把验证过的代码升级为可复用 capsule）是两件事。其二，system skills（记忆压缩、写回、反思、检索、GC）位于 Harness 内部 —— 它们维护 runtime 自身，因此不是面向用户的 Skill。

3Agent 是怎么工作的 — 分步走查

Harness 运行一个循环。由 LLM 做选择 —— 它制定计划、选下一个工具（O1）、为该步选模型（O7）、分配 token 预算（O8）—— 同时 typed contract、policy gate 与 verifier 保证它安全、不跑偏。这就是我们说的概率性控制面（命题 N0）：决策由 LLM 驱动，因此必须用审计日志 + 不变量来约束。

每一步先被检查再执行：在 Scaffold 上运行，或作为查询发往数据子系统，然后由 verifier/reward 打分。够好就交付产物；否则 agent 反思、修订计划并回到循环。后台里，system skills 在步骤之间持续保持记忆的精简与相关。

4Code 即契约 —— Harness 真正的引擎

深入看，Harness 不是「一堆工具」，而是一个以 code 为契约的可管理运行时。LLM 依 Skill 的要求产出 plan——子目标及其顺序，主体是自然语言、验收由 PRD（skill 的验收规约）控制。每个子目标可能有多个工具适用，于是 LLM 做概率性 fan-out：为竞争候选生成编排代码、派给子 agent（子 agent 之间的依赖也写在这段生成代码里），再verify 并筛选出最优。这就是 agent 循环。

这化解了一个表面矛盾：近期系统主张把调度器从 LLM 里移出到确定性内核。我们的回答（N6）：LLM 概率性地「写」代码、runtime 确定性地「跑」代码 —— 那段被生成、被验证的代码就是确定性内核。一个 reward-gated 的成功循环随后结晶为 Skill（N7）：其工具调用与结果因果被固化成一段 code-loop，可跨用户共享、作为单个 integrated tool 调用。因为 Skill 与 Harness 只经这段 code-loop 相遇，二者可各自演化（N8 / P10）：Skill 引入更强的工具；Harness 吸收成功 skill 并搭上越来越强的推理模型。

★可证伪的内核

C5

逻辑扩展与物理扩展正交，Harness 契约是解耦点。

P1

加 Skills 不拖垮吞吐（∂Θ/∂|S|≈0）；加 Scaffold 不改变覆盖（∂𝒯/∂|X|=0）。

N6

Code 即契约：LLM 概率性地写编排代码，runtime 确定、可审计地执行。

P10

Skill ⊥ Harness —— 只经 code-loop 耦合，两侧各自迭代（协同演化止于接口）。

N0

控制面是概率性的（LLM 驱动），靠审计日志 + 不变量使其可信。

P3

加 Skill 必须保持安全不变量 —— 单独 benign 的 skill 组合后可能有害。
