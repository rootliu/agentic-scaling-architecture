7 条架构精化

原话见 原始构想 I-3 ~ I-5。已融入主文档与形式化草稿。

S1

Skills 二分

task skills（L3）vs system skills（归 Harness）。判据：输出是否进入用户可见结果 + 触发源。

S2

Harness 内部三平面

契约翻译 + 能力供给（tool synthesis）+ 维护子系统（system skills）。

S3

维护子系统走物理轴

触发 ∝ token 流量、Haiku 级、指令固定 → 线性扩展 → 物理扩展。给出 P1 的机制解释。

S4

P1 可证伪前提

维护开销只依赖 token 量、不依赖 Skills 语义复杂度。可实验证伪。

S5

阈值参数化

θ_compact/θ_persist 由 KV Cache 经济学（显存→并发→最优 context）推出，非拍脑袋。

S6

SaaS/SSR 三轴解耦

三轴解耦是已知范式；delta = LLM 概率性下解耦的充要条件 + 可证伪判据。维护子系统 ≈ SSR 前端。

S7

ClawVM 一手证据

2604.10352 把 compaction/writeback/校验定位在 harness layer（"deterministic and auditable"）。引用时声明差异：ClawVM 偏确定性规则，本架构偏 LLM 驱动 + 可审计日志。

形式化与命题

A = ⟨S, H, X⟩ · P1 解耦 · P2 契约充分性 · P3 组合不变量

P1（修订）：逻辑扩展（+task skill）改变覆盖 𝒯，不改变单位 token 维护开销；物理扩展（+Scaffold + 配套廉价维护推理）改变吞吐 Θ（总成本随 token 线性增长），不改变 𝒯。
充要前提：维护开销只依赖 token 流量、不依赖 Skills 语义复杂度。
命题内容可证伪形式
P1∂Θ/∂|S|≈0 且 ∂𝒯/∂|X|=0（正交）固定 S 加 X 测 Θ（近线性）；固定 X 加 S 测 𝒯（Θ 不变）；前提：固定 token 变 Skill 多样性测压缩漂移
P2解耦充分条件 = 良定义契约 H，S 与 X 无直接依赖旁路 H → 观察是否退化为耦合
P3逻辑扩展保持安全不变量 Inv(S)⇒Inv(S∪{s})对标 SCR-Bench (2606.15242)：33.6% / 96.5% / +71.8%
