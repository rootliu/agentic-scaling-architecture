可扩展的 Agentic Runtime，以及它的可管理性。

读图顺序：先看中间三层 S/H/X 与 Harness 契约闸门，再看横向 CP/DP 职责分流；右侧 𝒟 是栈外数据协同系统，底部 P10–P13 解释并行/局部性，N8/P14 则把可验证 skill 固化为 code，作为概率控制面的确定性锚。

研究 agentic runtime 在持续增加 skills、执行资源、外部数据源和并行子 agent 时，如何仍保持可审计、可治理、可运维、可演进。扩展轴、控制/数据面、栈外数据子系统、并行/局部性是四条互相正交的切分；Skill-as-Code 提供确定性锚。

三层栈 ⟨S · H · X⟩ + 栈外数据子系统 𝒟

三个平面，一条契约；
外挂一个数据子系统。

自底向上三层各司其职，对应"虚机 / 框架运行时 / 前端运行时"。三层之外，再挂一个略独立的数据子系统——它不是第四层。

L3 · 逻辑轴

Skills

Specification Plane。specs 描述、I/O schema、调用 / 终止 / 循环条件。职责是组装并迭代推送进 LLM context。含 task skills。

 L2 · 契约

Harness

Capability / Contract Plane。把上层意图翻译为下层可执行单元。三分：契约翻译 + 能力供给（tool synthesis）+ 维护子系统 M。

 L1 · 物理轴

Scaffold

Execution / Isolation Plane。microVM / sandbox、隔离与算力。宽定义含推理 serving 容量，刻画系统 throughput Θ。
栈外 · 数据子系统 𝒟

Data Subsystem（不是第四层）

刻画 agent 如何理解、访问、治理、维护被授权的外部数据。沿在线/离线与 CP/DP 双重分裂：取数 D₁ 在数据面；语义总结 D₂ 是栈外独立 off-policy loop 𝓛₂（自带 loop / sub-agents / 模型）；治理 D₃、lifetime D₄ 在控制面。仅 semantic-join 查询接口进主路径。
核心论证
现代云应用早已"前端按流量横扩 / 业务按功能纵展 / 后端按算力扩"——多轴解耦是已知工程范式。 本研究的 delta：在每层内嵌概率性 LLM 调用的条件下，给出该解耦成立的充要条件（契约层 H）、可证伪判据（P1–P14），并证明四条正交切分复用同一不变量：高频在线路径与低频/离线/可预规划路径分离。

这把论文主张从"我们提出三层架构"提升为"我们证明在 LLM 不确定性下，逻辑 / 物理 / 数据 / 并行四类可扩展性可以解耦，且共享同一解耦机制；稳定技能还能固化为 code 以降低模型版本漂移"——回应"工程而非科学"的核心质疑。

创新点速览 四条切分

C5 + N0–N8，一张总表。

C5 为核心；N0–N8 沿控制面、数据子系统、并行局部性和 Skill-as-Code 展开，每条都绑定一个可证伪命题或实验设计。

C5 · 扩展轴

双扩展正交解耦

逻辑 / 物理扩展正交，解耦点=契约层 Harness。7 篇 2026 文献无一拆正交两维并绑不同层。→ P1–P3

 N0 · 平面

概率性控制面

LLM 驱动的控制决策（契约翻译 / 反思 / tool synthesis），需可审计日志 + 不变量约束。区别于确定性 SDN / K8s。→ P4–P6

 N1 · 数据

off-policy 语义总结

schema-on-read 后台预总结进不截断 lake，semantic join 按需注入。填补 Halevy(2605.28787) 的覆盖↔精度留空。→ P8

 N2 · 数据

数据治理即 skill

治理从"管边界"升级为"沉淀经验"：per-use-case data-usage skill。首次缝合数据治理与 agent skill / 记忆理论。→ P9

 N4 · 数据结构

off-policy loop 栈外独立

语义总结不是在线步骤或接口，而是正交于在线运行时的离线子系统（own loop / sub-agents / model）。≈ DB vacuum / 离线索引。

 N3 · N5

架构一致性 + skill 三分

N3：数据子系统沿 CP/DP 自然分裂，复用同一平面不变量。N5：skill 三分 task / system / data-usage，同遵 skill 形式。

 N6 · 并行局部性

工具集切分子 agent

用工具集闭包定义并行独立性、Skills 局部性和 Scaffold seed 边界，让并行度不再只是调度参数。→ P10–P12

 N7 · Dry Run

规划期探测可并行性

正式执行前先 dry-run 探工具集闭包与环境范围，再切分 sub-agent，降低跨 agent 重合度 ρ。→ P13

 N8 · Skill-as-Code

概率控制面的确定性锚

验证过且操作空间闭合的 skill 固化为 code，使输出与模型版本 θ 解耦，增强可审计、可复现和契约稳定性。→ P14

四维如何在一次运行里同时出现

同一条 spec 调用顺序，四条正交切分各自留下一阶足迹——这正是"共享同一解耦不变量"的运行时证据：

扩展轴 Skill 只声明 required_capabilities，H 翻译 𝓘→𝓔，从不直接绑 Scaffold（P2）。

控制/数据面 plan / 路由 / 验证在 CP，取数 / 计算 / serving 在 DP（P4–P6）。

数据子系统 先查 D₃/D₄/D₂ 共享摘要再取原文，源增加不抬单请求成本（P7）；写回逻辑追加、可回指原文防幻觉。

并行/局部性 dry-run 探工具集闭包再按不重合切分（P13/M1/M2），同 seed 共置（M4）、前缀冻结（P11）、只传摘要（M3）→ 吞吐逼近 1/s（P10）。

机制根因（§0.5 第一性原理）：一次推理 = 内在知识（权重，固定）⊕ context（可变），agent 唯一能动的是组织 context；前缀冻结 + 尾部变化同时买到缓存复用（快）、注意力不稀释（准）、并行独立（可切分）——P10–P13 即此根因的四个可证伪投影。
