可扩展的 Agentic Runtime，
以及它的可管理性。

论文的核心目标不是"三层栈"本身，而是研究 agentic runtime 在持续增加 skills、执行资源和外部数据源时，如何仍然保持可审计、可治理、可运维、可演进。扩展轴、控制/数据面、栈外数据子系统——三条互相正交的切分，共同服务于这个可管理的扩展目标。

三层栈 ⟨S · H · X⟩ + 栈外数据子系统 𝒟

三个平面，一条契约；
外挂一个数据子系统。

自底向上三层各司其职，对应"虚机 / 框架运行时 / 前端运行时"。三层之外，再挂一个略独立的数据子系统——它不是第四层。

L3 · 逻辑轴

Skills

Specification Plane。specs 描述、I/O schema、调用 / 终止 / 循环条件。职责是组装并迭代推送进 LLM context。含 task skills。

 L2 · 契约

Harness

Capability / Contract Plane。把上层意图翻译为下层可执行单元。三分：契约翻译 + 能力供给（tool synthesis）+ 维护子系统 M。

 L1 · 物理轴

Scaffold

Execution / Isolation Plane。microVM / sandbox、隔离与算力。宽定义含推理 serving 容量，刻画系统 throughput Θ。
栈外 · 数据子系统 𝒟

Data Subsystem（不是第四层）

刻画 agent 如何理解、访问、治理、维护被授权的外部数据。沿在线/离线与 CP/DP 双重分裂：取数 D₁ 在数据面；语义总结 D₂ 是栈外独立 off-policy loop 𝓛₂（自带 loop / sub-agents / 模型）；治理 D₃、lifetime D₄ 在控制面。仅 semantic-join 查询接口进主路径。
核心论证
现代云应用早已"前端按流量横扩 / 业务按功能纵展 / 后端按算力扩"——多轴解耦是已知工程范式。 本研究的 delta：在每层内嵌概率性 LLM 调用的条件下，给出该解耦成立的充要条件（契约层 H）、可证伪判据（P1–P9），并证明三条正交切分复用同一不变量。

这把论文主张从"我们提出三层架构"提升为"我们证明在 LLM 不确定性下，逻辑 / 物理 / 数据三种可扩展性可以解耦，且共享同一解耦机制"——回应"工程而非科学"的核心质疑。

C5 · 扩展轴

双扩展正交解耦

逻辑 / 物理扩展正交，解耦点=契约层 Harness。7 篇 2026 文献无一拆正交两维并绑不同层。→ P1–P3

 N0 · 平面

概率性控制面

LLM 驱动的控制决策（契约翻译 / 反思 / tool synthesis），需可审计日志 + 不变量约束。区别于确定性 SDN / K8s。→ P4–P6

 N1 · 数据

off-policy 语义总结

schema-on-read 后台预总结进不截断 lake，semantic join 按需注入。填补 Halevy(2605.28787) 的覆盖↔精度留空。→ P8

 N2 · 数据

数据治理即 skill

治理从"管边界"升级为"沉淀经验"：per-use-case data-usage skill。首次缝合数据治理与 agent skill / 记忆理论。→ P9

 N4 · 数据结构

off-policy loop 栈外独立

语义总结不是在线步骤或接口，而是正交于在线运行时的离线子系统（own loop / sub-agents / model）。≈ DB vacuum / 离线索引。

 N3 · N5

架构一致性 + skill 三分

N3：数据子系统沿 CP/DP 自然分裂，复用同一平面不变量。N5：skill 三分 task / system / data-usage，同遵 skill 形式。
