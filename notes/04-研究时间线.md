## 主线演进（叙事）

**阶段一·立论（M1–M6）**：从"研究 agentic runtime"出发，提炼出三层参考架构与双扩展维度，锁定 C5 为 novelty 内核，并用 7 篇 2026 文献交叉验证其空白性。

**阶段二·夯实（M7–M10）**：把讨论沉淀进 Obsidian 与 Zotero；盘点已有 vault 素材并完成二手数据可引用性核实——确立"科学性靠自复现实验、不靠二手引用"的方法论底线。

**阶段三·精化（M11–M13）**：用户补充"system skills 属 Harness + 维护子系统走物理轴 + SaaS 前端类比"，使 C5 从"分层主张"升级为带机制解释与可证伪前提的命题；产出 sub-agents 目录与可视化 worktree。

**阶段四·二维化（M14）**：在"加什么"（逻辑/物理扩展）之外，叠加"怎么分流"（控制面/数据面）的正交切分，使 C5 升级为二维设计框架；P1 获得第二个结构性论据（作用平面不同），并提出 agent 系统特有的"概率性控制面"为新贡献点。

## 关键判断的演化

| 判断 | 早期 | 现在 |
|---|---|---|
| C5 性质 | "分层架构"（易被批工程） | 三轴解耦的充要条件 + 可证伪判据（SaaS delta） |
| Skills | 同质集合 | task / system 二分 |
| Harness | 契约翻译层 | 契约翻译 + 能力供给 + 维护子系统 |
| 物理扩展 | 加 sandbox | 加 Scaffold（隔离+算力）+ 配套廉价维护推理 |
| 10× 数据 | 拟直接引 | 仅动机 + 自复现（二手转引规避） |
| 架构维度 | 一维：逻辑/物理扩展轴 | 二维：扩展轴 × 控制面/数据面（2×3 矩阵）|
