## 0. 一句话总纲

> 我们提出一个 agentic runtime 的**三维参考架构**，并证明三条正交切分（**扩展轴 × 控制/数据面 × 数据子系统**）复用同一组解耦不变量；核心主张是 agent 的**逻辑可扩展性、物理可扩展性、数据可扩展性可以解耦**，解耦点是层间显式契约（Harness）与异步独立子系统（off-policy loop / 治理 memory）。我们给出形式化、可证伪命题（P1–P9）与实验设计。

## 2. 四条正交切分如何复用同一解耦不变量（架构一致性论证）

| 切分 | "加什么" | "怎么分流" | 解耦命题 | 解耦点 |
|---|---|---|---|---|
| 扩展轴 | 逻辑(+Skill) / 物理(+Scaffold) | — | P1 | 契约层 $H$（P2） |
| 控制/数据面 | — | CP(决策, ∝事件) / DP(token, ∝流量) | P4/P5 | 平面分离公理 |
| 数据子系统 | 数据(+src) | 在线($D_1$/$D_2$查询) / 离线($\mathcal{L}_2$/$D_3$/$D_4$) | P7 | 异步独立 loop + 独立 API |
| 并行/局部性 | 同种子 agent(+实例) | 内聚(同 seed 的 context/资源) / 松散(跨 agent 摘要传递) | P10 | 工具集切分 + seed 亲和 + 摘要接口 |

> **统一命题（拟写进 paper 的 meta-claim）**：三维各自的"加东西"操作，其一阶成本都**落在与主请求路径解耦的平面/子系统**上——逻辑扩展落控制面、物理扩展落数据面、数据扩展落离线 off-policy loop。故三维**共享同一个解耦机制：把高频在线路径与低频/离线决策路径分离**。这是把"三层工程架构"提升为"可证明科学主张"的总杠杆。

## 3. 可证伪命题总表（P1–P13）

| 命题 | 一句话 | 证伪条件 |
|---|---|---|
| P1 解耦 | $\partial\Theta/\partial\|S\|\approx0$ 且 $\partial\mathcal{T}/\partial\|X\|=0$ | 加 Skills 显著拖垮 throughput |
| P2 契约充分 | 良定义 $H$ ⇒ S/X 无直接依赖 | 旁路 $H$ 后仍解耦 |
| P3 组合不变量 | $\text{Inv}(S)\Rightarrow\text{Inv}(S\cup\{s\})$ | 组合后越权/不变量破坏 |
| P4 平面分离 | $\partial L_{CP}/\partial\dot\tau\approx0$ | QPS↑ 使控制开销同比上升 |
| P5 P1 平面分解 | 逻辑→CP、物理→DP | 二者作用同一平面 |
| P6 M 跨面 | M 触发在 CP、开销在 DP | M 触发频率 ∝ 业务流量 |
| **P7 取数解耦** | 固定 use case，$\partial\ell/\partial\|\text{src}\|\approx0$ | 加数据源抬高单请求 token 成本 |
| **P8 摘要充分性** | off-policy $\Sigma$ 兼顾覆盖与精度 | (c) 摘要方案不优于 (a)在线/(b)人工 |
| **P9 治理收敛** | data-usage skill 使取数决策熵单调不增 | 重复 use case 决策发散/震荡 |
| **P10 并行可设计** | 按工具集不重合切分($\rho\downarrow$)+摘要传递($E\uparrow$)使吞吐逼近 $1/s$ | 高/低 $\rho$×全量/摘要 无差异 |
| **P11 工具集局部性** | 工具集稳定+前缀逐token冻结→context 命中 $H\uparrow$ | 工具集稳定性/前缀冻结与缓存命中无关 |
| **P12 seed 亲和** | 同 seed 共置+异 seed 分散→固定资源下 $k_{\max}\uparrow$ | seed 亲和放置无物理局部性收益 |
| **P13 规划期 dry-run** | 规划期探测工具集闭包/环境范围再切分→$\rho\downarrow$、$H\uparrow$ | dry-run 切分不优于纯语义切分 |
| **P14 跨模型稳定** | code 固化 skill 输出与 θ 版本解耦 ∂output/∂θ≈0 | 跨版本重跑 code skill 方差不低于纯 LLM skill |
| **P15 双子目标分解** | $r_{\text{out}}\times r_{\text{proc}}$ 双维 reward 比单一 scalar 归因更清晰、收敛更快、人工干预更少 | 双维分解相较单一 scalar 无收敛速度/人工干预优势 |
| **P16 还原式 reflection** | 若 agent 能从 $\Sigma(\text{src})$+参照报表规范还原出参照报表关键数值/结构，则 $\Sigma$ 对该 use case 充分 | 还原准确率长期与直接读原表存在固定差距 |
| **P17 IR 松耦合** | 数据源变更与产出格式变更互不传播，只需更新 IR 记录 | 任一侧变更扩散进另一侧 wiki 的内部结构 |

## 4. 与 related work 的划界（防 reviewer "已有人做过"）

- **vs 数据发现/语义层**（CoeusBI 2606.15384、Pneuma 2601.03618、Octopus 2601.02304、EasyTUS 2511.02674）：它们**单点**（建视图/表发现/table-union），无 latent 保留承诺、无独立 off-policy loop、无治理 memory + lifetime 闭环 → N1/N4 区别。
- **vs 语义元数据必要性**（Halevy 2605.28787）：它论证"要不要元数据"并暴露覆盖↔精度二难；我们提出**第三条路**（off-policy 预总结）并以 P8 填空。
- **vs 数据治理**（Five-Plane 2606.12320、DataGovBench 2512.04416、DPDP 2601.01101）：它们做 enforcement/合规；我们把治理**升级为 skill/memory 沉淀**（N2/P9）。
- **vs agent 记忆**（记忆演化综述 2605.06716）：它讲 agent **自身轨迹**记忆；我们讲**外部数据使用**的治理记忆 → 正交补充。
- **vs serving/runtime layer**（Policy-Driven 2605.27744、Helium 2603.16104）：它们优化在线 serving；我们的 off-policy loop 正交于在线路径（N4）。

## 5. 写 paper 时的 Contributions 段（草稿措辞）

> We make five contributions. **(C1)** We formalize agent extensibility as two orthogonal axes—logical (Skills) and physical (Scaffold)—and prove their decoupling holds iff an explicit contract layer (Harness) mediates them (P1–P3). **(C2)** We show this runtime admits the classical control/data-plane cut, but with a *probabilistic control plane* whose LLM-driven decisions require auditable logs and invariant enforcement (P4–P6). **(C3)** We introduce an agent-native data subsystem with an *off-policy semantic-summary loop*—an independent, asynchronous batch process (own loop, sub-agents, even its own model) that pre-summarizes sources schema-on-read into a non-truncating lake, retrieved via semantic join (P7–P8); this fills the coverage-vs-precision gap left open by Halevy et al. **(C4)** We recast data governance from boundary enforcement to *experience accumulation*, recording per-use-case data-usage as reusable skills that converge over repetition (P9), unifying data governance with agent skill/memory theory. **(C5)** We show all three orthogonal cuts reuse one decoupling invariant—separating high-frequency online paths from low-frequency/offline decision paths—evidencing architectural coherence.
