## 0. 一句话总纲（融合后）

> ICA 与本架构是**同一现象的两种语言**：ICA 说"LLM 是概率 CPU、agent runtime 是新操作系统"，本架构说"⟨S,H,X⟩ 三层栈沿契约 H 解耦、外挂栈外数据子系统 𝒟"。两者可**逐层对齐**；其唯一实质分歧——控制面是"确定性"还是"概率性"——恰恰是本架构 novelty **N0（概率性控制面）** 的立论点，ICA 的"确定性控制平面"应被修正为"**确定性护栏 + 概率性决策**"。ICA 的独有价值在于把定性设计空间**定量化**（局部性 / 上下文预算 / 代理加速三条阿姆达尔式启发式），可直接作为 P1/P4/P7 的机制解释与可测形式。

## 1. ICA 六层 ↔ 本架构三层栈 + 数据子系统（核心映射）

ICA 的六层功能层次，与本架构的 ⟨S,H,X⟩ + 𝒟 并非竞争，而是**粒度不同的同一栈**。ICA 把本架构的 **Scaffold（物理执行层 X）拆成 L1/L2**，把 **Harness（契约层 H）拆成 L4/L5**，把 **上下文管理单列为 L3**：

| ICA 层 | 职责（ICA 语言） | 本架构映射 | 平面归属（本架构 §05） |
|---|---|---|---|
| **L1 物理执行** | GPU/TPU 执行 matmul/attention，Token Batch → Logits | **Scaffold X**（宽定义含 serving 算力） | 运行时数据面 DP |
| **L2 推理服务** | 权重加载、KV 缓存、连续批处理、prefill-decode 调度（vLLM PagedAttention） | **Scaffold X** 的 serving 子层 | DP（执行）+ CP（调度放置） |
| **L3 上下文管理** | 热/暖/冷分层、压缩、摘要、检索（MemGPT） | **Harness 维护子系统 $M$** + **数据子系统 $D_1/D_2$ 查询接口** | 触发在 CP、token 开销在 DP（= P6） |
| **L4 语义接口** | Tool ABI、memory API、协议 MCP（"AI 的 USB-C"） | **契约层 $H:\mathcal{I}\to\mathcal{E}$**（能力供给 / tool schema） | 运行时控制面 CP |
| **L5 编排** | 任务规划、代理调度、权限强化、故障恢复、状态提交 | **契约层 $H$ 的控制逻辑** + **不变量执行 Inv** | 运行时控制面 CP |
| **L6 应用** | 最终用户任务与领域 agent 应用 | **Skills 层 S**（task skills）+ 领域 use case | L3 Skills 的 DP/CP（见 §05 矩阵） |
| **（ICA 未单列）** | agent 与外部数据的理解/访问/治理/维护 | **栈外数据子系统 𝒟 = ⟨D₁,D₂,D₃,D₄,Ω⟩** | 双重分裂：$D_1$∈DP、$D_2$=栈外 off-policy loop、$D_3/D_4$∈CP |

> **对齐要点**：
> - ICA 把本架构的 **X（Scaffold）细分为 L1+L2**——这正呼应本架构"Scaffold 宽定义 = 隔离 + 算力（含 serving）"。ICA 的 L2/PagedAttention 是 X 内部的微架构层。
> - ICA 的 **L3 上下文管理 = MemGPT 式虚拟内存**，在本架构里被拆到两处：**运行时侧**是 Harness 维护子系统 $M$（压缩/写回），**外部数据侧**是数据子系统 $D_1/D_2$。ICA 用一个"虚拟内存"隐喻统一了两者，本架构则按"运行时自身 vs 外部数据"作更细区分（对应 N4/N5 的 skill 三分）。
> - ICA 的 **L4+L5 = 本架构的契约层 H**。ICA 把"接口契约（ABI/系统调用）"与"内核编排（调度/权限）"分成两层，本架构统一在 H 里并用 CP/DP 平面切分表达（§05）。
> - ICA **没有单列数据子系统 𝒟**——这是本架构相对 ICA 的**增量**（N1–N4）。ICA 的 L3 只谈"上下文/外部存储虚拟内存"，未识别出"独立 off-policy 语义总结 loop $\mathcal{L}_2$"这一栈外异步子系统。

## 2. 双平面架构：ICA vs 本架构的关键冲突与消解

ICA 提出**概率执行平面 + 确定性控制平面**，用以解决"LLM 是 CPU 还是 OS"之争。本架构 §05 提出**运行时 DP + 运行时 CP**，并明确 novelty **N0 = 概率性控制面**。二者在"执行平面"上一致，但在"控制平面"上**直接冲突**：

| | ICA 简报 | 本架构（[05-控制面与数据面正交切分](05-%E6%8E%A7%E5%88%B6%E9%9D%A2%E4%B8%8E%E6%95%B0%E6%8D%AE%E9%9D%A2%E6%AD%A3%E4%BA%A4%E5%88%87%E5%88%86.md)） |
|---|---|---|
| 执行平面 | 概率执行平面（推理生成，概率性、上下文敏感、无持久态） | 运行时数据面 DP（token 处理路径，∝ 流量） — **一致** |
| 控制平面 | **确定性**控制平面（权限/审计/策略/恢复，必须确定可预测可审计） | **概率性**控制面 N0（契约翻译/反思/tool synthesis **由 LLM 驱动**，决策本身不确定） — **冲突** |

**消解（本架构立场，须写进 paper 以彰显 N0）**：
ICA 把控制平面设为"确定性"，是沿用 SDN/K8s 的经典假设。但 agentic runtime 的控制决策——契约翻译 $H$、$M$ 的反思触发、tool synthesis——**本体是 LLM 调用，天然概率性**。因此正确的划分不是"概率执行 / 确定控制"，而是：

> **概率性决策 + 确定性护栏**：控制平面里，**决策的产生**是概率性的（LLM 生成计划/翻译/工具），但**决策的执行门控**是确定性的（权限检查、不变量 Inv、审计日志、审批门）。ICA 的"确定性控制平面"应精化为"控制平面的 **enforcement 子层**确定，**decision 子层**概率"。

这恰好把 ICA 的"任何副作用操作须经确定性平面审批门控"与本架构的"用可审计日志 + 不变量执行约束概率性控制面"**缝合**：二者说的是同一件事的两半——ICA 强调"门"（enforcement 确定），本架构强调"门后的决策器"（decision 概率）。**合并结论：控制平面 = 概率决策器 ⊕ 确定护栏门**，这是比 ICA 原文更准确的 N0 表述。

> **进一步精化（2026-07-12，见 [10-Skill-as-Code与确定性固化](10-Skill-as-Code%E4%B8%8E%E7%A1%AE%E5%AE%9A%E6%80%A7%E5%9B%BA%E5%8C%96.md)）**：上面的二分只回答了“校验确定”（护栏），未回答“生成确定”从哪来。skill-as-code 补上第三层：当一个 skill 操作空间闭合时，把其过程固化为 code，则这段执行不再调 θ、不再采样。故 **控制平面 = 概率决策器 ⊕ 确定执行体（code-frozen skill）⊕ 确定护栏门**；A3（概率执行公理）的作用域从“整个控制面”收窄到“未 code 化的决策部分”。这给 §6 协作悲论（委派率 0–20%）第二个抓手：**可 code 化的部分先确定下来，人只需审概率部分。**

> 关联证据：ClawVM 把 compaction/writeback 放在 harness layer 但偏确定性规则；本架构偏 LLM 驱动 + 可审计日志（见 [03-Harness-System-SubAgents清单](03-Harness-System-SubAgents%E6%B8%85%E5%8D%95.md)）。ICA 的 L5"权限强化/故障恢复"= 本架构的确定护栏门。

## 3. ICA 六大公理 ↔ 本架构 P1–P9 命题（机制补强）

ICA 的六条设计公理，多数是本架构命题的**机制性前提或定性表述**。把它们接入 P 系列，可为命题提供"为什么成立"的直觉：

| ICA 公理 | 内容 | 本架构对应 | 关系 |
|---|---|---|---|
| **A1 局部性** | 计算有语义局部性，近期上下文极可能再访问 | 支撑 §4 启发式 I（KV 缓存命中）；机制上支撑 P1（物理扩展靠缓存重用不拖垮 throughput） | ICA 提供机制，本架构提供可证伪判据 |
| **A2 分层抽象** | 每层只依赖下层接口非实现 | = 契约层 $H$ 的充分性 **P2**（S/X 经 $H$ 无直接依赖） | 同一主张，A2 定性 / P2 可证伪 |
| **A3 概率执行** | 系统须容忍随机性，以统计质量边界 + 语义验收替代确定性验证 | = N0 概率性控制面的**必要前提**；对应 P4–P6 需"可审计性指标" | A3 是 N0 的公理化 |
| **A4 虚拟化** | 用分层存储制造"无限上下文"幻觉 | = L3 上下文管理 = 维护子系统 $M$ + 数据子系统 $D_1/D_2$；支撑 §4 启发式 II（上下文预算） | A4 是 $M$ 与 $\mathcal{D}$ 存在的公理化理由 |
| **A5 最小权限** | 每个执行单元仅得完成任务的最小能力令牌 | = **P3 组合不变量** $\text{Inv}(S)\Rightarrow\text{Inv}(S\cup\{s\})$ 的实现手段；= 数据治理 $D_3$（N2） | A5 是 P3/N2 的工程约束 |
| **A6 可观测性** | 每个副作用操作须产出可追踪可审计事件 | = N0 概率性控制面的**约束手段**（可审计日志）；= §2 确定护栏门的日志侧 | A6 使 N0 可控 |

> **写作杠杆**：A1–A6 可作为 paper 的"设计原则"小节，逐条落到 P1–P9 与 N0。它们把本架构从"我们证明 X"补上"为什么应该这样设计（体系结构公理）"，回应 reviewer 对动机的追问。

## 4. 定量设计启发式（ICA 独有增量，接入 P1/P4/P7）

ICA 最有价值的增量是三条**阿姆达尔定律风格**的启发式，把本架构定性的"解耦"变成**可测量的工程直觉**。建议直接纳入本架构的实验设计（[05-控制面与数据面正交切分](05-%E6%8E%A7%E5%88%B6%E9%9D%A2%E4%B8%8E%E6%95%B0%E6%8D%AE%E9%9D%A2%E6%AD%A3%E4%BA%A4%E5%88%87%E5%88%86.md) §7、[06-数据平面四组成架构](06-%E6%95%B0%E6%8D%AE%E5%B9%B3%E9%9D%A2%E5%9B%9B%E7%BB%84%E6%88%90%E6%9E%B6%E6%9E%84.md) 的命题验证）。

### 启发式 I · 语义局部性（接 P1 物理扩展）

$$S = \frac{1}{(1-H) + H\cdot\alpha^{-1}}$$

其中 $H$=KV 缓存命中率，$\alpha$=重用加速因子。**设计启示**：命中率提升边际收益递增；多轮对话中提升缓存重用比单纯提速更具性价比。
**接入本架构**：这是 **P1**（$\partial\Theta/\partial|S|\approx0$）在 serving 层的机制——物理扩展之所以不被逻辑扩展拖垮，部分靠语义局部性带来的缓存重用（A1）。也是"语义陈旧性（semantic staleness）导致缓存失效"这一未来挑战的量化入口。

### 启发式 II · 上下文预算（接 P5 / A4 虚拟化）

$$W_{\text{eff}} = \int_{0}^{C}\beta(L)\,dL$$

$C$=物理窗口长度，$\beta(L)$=位置 $L$ 的注意力保留率。**设计启示**："物理窗口长度 ≠ 有效记忆容量"；系统设计应通过摘要/检索提高平均 $\beta$。
**接入本架构**：这正是**维护子系统 $M$**（压缩/写回）与**数据子系统 $D_2$**（off-policy 语义总结）存在的定量理由——它们的目标函数就是"在固定 $C$ 下抬高 $W_{\text{eff}}$"。为 **P8（摘要充分性）** 提供可测目标：好的 off-policy 摘要方案应显著增大 $\int\beta$。

### 启发式 III · 代理加速（接 P4/P6 + C5 物理扩展）

多代理效率受任务**串行比例**与**编排效率系数 $E$** 限制；盲目加 agent 因通信开销收益递减（阿姆达尔上界）。
**接入本架构**：这是 **P4（平面分离）** 与 **P6（$M$ 跨面）** 的量化边界——编排开销 $E$ 属**运行时控制面**，若 $E$ 随 agent 数劣化，说明控制面被通信拖入数据面（P4 反模式）。也界定了 C5 物理扩展的**上界**：Scaffold 横扩的收益受编排串行比约束。

## 6. 工业证据（ICA 提供，接入 [04-研究时间线](04-%E7%A0%94%E7%A9%B6%E6%97%B6%E9%97%B4%E7%BA%BF.md) 与动机段）

- **Rakuten / Claude Code**：在 1250 万行代码库上自主运行数小时（支撑 C5 物理扩展的规模可行性 + L5 编排层实证）。
- **CTO 4–8 月项目 → 2 周**：开发周期从"周"压到"小时"级（L6 应用层价值）。
- **协作悖论**：AI 使用率约 60%，但完全委派率仅 0–20%。**本架构解读**：这正是 **N0 概率性控制面**要解决的问题——委派率上不去，因为缺"确定护栏门"（§2 的 enforcement 子层）给出安全保证。**确定护栏门的完备性是提高委派率的前提** → 为 P3（组合不变量）/ N2（治理即 skill）提供工业动机。
- **约 30% AI 代码建议被接受**：概率执行的统计质量边界（A3）的现实刻度。

## 7. 融合后的 novelty 边界更新（防"ICA 已提出"）

ICA 简报本身是**类比框架与设计哲学**（六层 + 双平面 + 六公理 + 三启发式），**不含可证伪命题、不含数据子系统、不区分概率/确定控制面的子层**。因此本架构相对 ICA 的 delta 明确：

1. **N0 更精确**：ICA 的"确定性控制平面"→ 本架构"概率决策器 ⊕ 确定护栏门"（§2）。
2. **𝒟 全新**：ICA 无独立数据子系统与 off-policy loop $\mathcal{L}_2$（N1/N4）。
3. **可证伪**：ICA 的公理/启发式是定性/直觉，本架构给出 P1–P9 的证伪条件与实验。
4. **skill 三分**（N5）与**治理即 skill**（N2）：ICA 的 L4/L6 未涉及。

> 反过来，本架构应**吸收** ICA 的：六层细粒度（§1，把 X/H 拆细以对齐工业栈 vLLM/MCP/Codex）、六公理作为设计原则（§3）、三启发式作为可测机制（§4）、工业证据作为动机（§6）。这使论文既有"科学主张（P1–P9）"又有"体系结构叙事 + 定量直觉"，两条腿走路。
