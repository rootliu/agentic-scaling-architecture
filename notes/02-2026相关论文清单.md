### 简要总结（précis）

**R1 · Do Agents Need Semantic Metadata?（2605.28787, Halevy 等）** — 比较研究：LLM agent 做数据发现/取数时，究竟需不需要预建的语义元数据，还是裸跑非结构化 web 内容即可。实证结论：FAIR 语义生态使取数 precision **+65.7%**，但**覆盖受限**；裸 agent 在线发现覆盖广却 **last-mile 精度崩**。**与本架构**：这是 $D_2$/P8 的**直接靶子**——该文暴露"覆盖↔精度"二难并**明确把"后台 off-policy 预总结"列为留空方向**，我们的 N1（schema-on-read 语义总结平面）+ P8（摘要充分性）正是填这条折中路。**delta**：它只比较"要/不要元数据"两端，未提出第三条 off-policy loop 路径。

**R2 · CoeusBI（2606.15384, 百度）** — 工业级交互式 BI 系统：双 agent + 层级 schema linking，处理复杂 SQL 与超宽 schema，在百度落地、提升多轮对话准确率与 token 效率。**与本架构**：最接近 $D_2$ 的"离线自动建语义视图"。**delta**：仍是**单点**（建视图/schema linking），无 latent-field 保留承诺、无独立 off-policy lake、无治理 memory + lifetime 闭环。

**R3 · Data Product MCP（2601.08687）** — 基于 MCP 的企业数据 marketplace：让 agent 发现并查询企业数据，实时强制治理策略，集成 Snowflake/Databricks，AI 驱动的访问控制。**与本架构**：$D_1$ on-policy 取数 API 的工业实现。**delta**：取数 + access control 单点，未与"off-policy 预总结 + 治理 memory + lifetime"统一。

**R4 · DataGovBench / DataGovAgent（2512.04416）** — 150 个真实数据治理任务基准 + DataGovAgent（planning + RAG + 沙箱 debug），降低多步 workflow 的 debug 迭代。**与本架构**：$D_3$ 治理的最接近 baseline。**delta**：治理=合规 enforcement/任务完成度，**未把"如何用数据"沉淀为可复用 data-usage skill**（→ N2/P9 是新的）。

**R5 · EasyTUS（2511.02674）** — 基于 LLM 的 table-union search 框架：模块化 serialization + representation + 向量检索，在缺元数据场景仍鲁棒，precision/速度双升。**与本架构**：可作 $D_2$ 的 **semantic join 检索** baseline / 公开基准数据。**delta**：只解 table-union 这一单点检索，无总结产物、无治理。

**R6 · Octopus（2601.02304）** — 免训练、实体感知的多表数据发现 + cell-level 检索：从 query 抽细粒度实体匹配表头，避开重离线处理，低成本高精度。**与本架构**：同为 $D_2$ 检索 baseline / 公开数据。**delta**：训练-free 在线发现路线（恰是 Halevy 说的"覆盖高精度低"那端），无 off-policy 预总结。

**R7 · Pneuma（2601.03618）** — 把"信息需求"具象为关系 schema：迭代交互逐步精化数据模型，集成 RAG + agentic，挖掘 latent intent、沉淀组织知识。**与本架构**：跨 $D_2$（schema 化）与 $D_3$（组织知识沉淀）。**delta**：以**人在环交互**精化为主，非后台独立 off-policy loop；知识沉淀未形式化为 per-use-case skill + 收敛命题。

**R8 · DAR / Beyond Text-to-SQL（2512.14622）** — 多 agent 自主端到端数据库研究（无需用户 prompt）：initialization/execution/synthesis 三层全在云数仓内编排，比人工分析快约 **32×**，产证据 grounded 洞察。**与本架构**：$D_1$ 之上的自主取数+分析 agent。**delta**：聚焦"自动跑分析"，无语义总结 lake、无治理记忆/lifetime 维度。

**R9 · Agentic Data Governance under DPDP（2601.01101）** — 面向印度 DPDP 隐私合规的 agent 框架：语义理解 agent + 合规 agent，跨行业透明执行领域特定的匿名化策略。**与本架构**：$D_3$ 的合规 enforcement 实例。**delta**：纯合规/隐私 enforcement，不涉及"数据使用经验沉淀为 skill"。

> **统一划界**：上述 9 篇分别覆盖取数（R3/R8）、语义发现/总结（R2/R5/R6/R7）、必要性论证（R1）、治理（R4/R9）的**单点**；**无一**把它们统一为一个含**独立 off-policy loop（$\mathcal{L}_2$，栈外）+ 治理即 skill（$D_3$/P9）+ lifetime（$D_4$）**的 agent-native 数据子系统，也无一沿运行时 CP/DP 复用同一平面不变量（N3）。这构成 $\mathcal{D}$ 的 novelty 边界，详见 [07-创新点总结](07-%E5%88%9B%E6%96%B0%E7%82%B9%E6%80%BB%E7%BB%93.md)。
